Flask 源码阅读: 启动流程

Posted on 2018-09-09 | In Flask |

阅读 Flask 源码的系列笔记, 这一篇说的是 Flask 的启动流程

这里是用 Debug 模式运行所以出现的会是 DebugApplication 对于其他情况都会调用不同的 Application. 例如, 多线程是 ThreadedWSGIServer, fork 是 ForkingWSGIServer等, 但是除了具体等 多线程 和 fork 操作不一样之外, 其他等流程大致也是一样的. 就不详细写了.

以后有时间在看看要不要上图之类的.

Flask 启动流程

启动代码

Cli 启动

  1. 开始启动后, 首先执行 flask/main.py 文件
    这个文件是 Flask.run 在命令行中等别名, 它会导入 flask.cli 并执行 cli.main 方法

  2. cli.main 方法会重写参数方便重启, 然后执行 FlaskGroup 实例等 main 方法
    在进入 cli.py 后会实例 FlaskGroup 赋值给全局变量 cli .然后执行 cli 的 main 方法.
    FlaskGroup 是 AppGrooup 的特殊子类, 一般不会用到, 但是有些用例需要用到.
    AppGroup 是 Click.Group 的子类重新封装了 command 装饰器以便将函数封装到 with_appcontext

  3. cli.main 会设置一个全局标志, 然后启动命令

    1. cli.main 设置一个全局标志 FLASK_RUN_FROM_CLI 为 ‘true’, 表明是从命令行调用到.
      这个标志是被 Flask.run 检测, 使调用进入无操作状态.当这里当加载脚本也尝试启动服务器使, 可以避免一些错误.
    2. cli.main 还会在这里检测是否加载 dotenv. 设置系统变量 FLASK_SKIP_DOTENV 判断是否加载.
    3. 这个时候会创建 ScriptInfo 实例.
      ScriptInfo 是帮助处理 Flask applications 的对象. 但是通常是可以忽略的的,因为它在调度中用于内部 click.
      在以后的版本可能会变得重要. 一般是由 FlaskGroup 创建, 但是也可以手动创建并作为 click 对象传递.
    4. 会调用父类 BaseCommand 的 main 方法, 执行 Flask.cli.run_command 方法启动服务器.
      BaseCommand 是 click 实现的的最小 API 命令契约. 通过 Click 调用 Flask.cli.run_command 方法.
      ScriptInfo 实例将作为参数 info 传递进 run_command 方法, 其他一些运行参数也会被传递进 run_command 方法里,
      例如: host, port, reload, debugger, eager_loading, with_threads, cert等.
      其中 cert 是用于指定 HTTPS 证书, eager_loading 是控制是否快速加载的, with_threads 是用于是否启用多线程的.
      cert -> –cert
      eager_loading -> –eager-loading/–lazy-loader
      with_threads -> –with-threads/–withour-threads

  4. run_command 方法会创建一个本地开发服务器.

    1. 设置第一次运行服务器时显示的额外启动信息
    2. 实例化 cli.DispathingApp 赋值给 app, 在实例化 Flask 的过程中会初始化 Request、Response、config 等配置给 app
      DispathingApp 是一个在后台线程中通过 name 导入模块, 然后分发到 Flask Application 到特殊应用。
      如果发生错误，它会被记录并显示未 WSGI handling 的一部分。
      1. 首先获取多线程锁 threding.Lock()
      2. 根据是否启用快速加载(–eager-loading/–lazy-loader)来选择不同的加载方式 Flask_APP -> export FLASK_APP=xxx.py
        如果启用则使用 _load_unlocker() 加载, 否则则使用 _load_in_background() 加载
        • _load_unlocker()
          1. _load_unlocker 会调用 self.loader() -> ScriptInfo.load_app
            这个方法会加载 Flask app 并返回, 重复调用这个方法会返回已加载的 app
        • _load_in_background
          1. _load_in_background 会使用多线程的方式调用 _load_app 在获得线程锁的时候再通过 _load_unlocker
    3. 导入 werkzeug.serving.run_simple 方法创建服务器.

  5. werkzeug.serving.run_simple 启动一个 WSGI application

    1. debug 模式和静态文件加载是在这里被添加进 application
    2. 如果允许模块修改时自动重启 Python 进程, 会提前创建 socket 保证端口可用.
    3. 最后都是调用 inner 方法创建服务器

  6. inner

    1. 调用 make_server 创建一个新的 server 实例 srv
      1. make_server 可以创建阻塞、多线程、多进程等多种实例
      2. 在创建 WSGI server 的时候会初始化 HTTPServer
    2. 通过 srv.serve_forever 启动服务

app.run 启动

  1. 与 Cli 启动不同的是 app.run 启动会在先初始化 Flask 给 app.在初始化 Flask 的过程中会和 Cli 启动一样会初始化配置.

  2. app.run() 启动一个本地服务器

    1. 在 app.run() 里面会用到 cli.load_dotenv() 加载 dotenv,这个 cli 是在初始化时 通过 from . import cli 导入的.
